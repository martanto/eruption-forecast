Two-step. `TremorMatrixBuilder` joins the tremor CSV to the label window grid, validating per-window sample counts and producing a long-form matrix keyed by `(id, datetime)` with one column per tremor metric. `FeaturesBuilder` then runs tsfresh feature extraction column-by-column (training mode filters windows to those present in the label CSV; `use_relevant_features` may apply tsfresh's FDR filter). The result is concatenated into a unified feature matrix joined back to labels by window id.
